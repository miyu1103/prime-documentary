# Motion Quality Gate（モーション品質ゲート）

ステータス: 設計（draft）

前提・仮定:

- 本書は CANON（登録済みツールのみ前提）に従う。前提ツール: Claude(Pro/Max), ChatGPT(Plus/Pro), GitHub, Runway, ElevenLabs, Midjourney, SUNO, LottieFiles(無料登録)。新規有料ツール・素材サブスクは前提にしない。
- ツール役割: Midjourney=静止画/パーツ, Runway=見せ場動画のみ, ElevenLabs=ナレ+SFX, SUNO=BGM, Lottie=手動DLしたローカルJSONのみ。
- 本書は `docs/12_QUALITY_GATES_AND_ACCEPTANCE.md`（以下「12」）の**下位・特化ゲート**である。12が全工程の合否を定義し、本書はそのうち Visual Gate / Edit Gate に入る「モーション（動き・アニメーション）」部分を細分化して判定する。12と矛盾する判定はしない。12のSeverity（S0〜S5）をそのまま使う。
- 「20 sceneType」「30 motion grammar」はCANON共通語彙であり、本ゲートはそれらが正しく選択・適用されているかを評価する（個別の20/30の一覧定義は別CANON文書に従い、本書では参照のみ）。
- 「props・preset」とは Remotion/コード側でモーションを差し替え可能にするパラメータ（持続時間、イージング、方向、強度、レイヤー順など）を指す。
- 「Asset Factory / 棚（Asset First）」とは、先に蓄積した再利用可能素材（Midjourney静止画、ElevenLabs SFX、SUNO BGM、Lottie JSON、Runway見せ場クリップ）の在庫を指す。

---

## 1. このゲートの目的

「Generated successfully ≠ usable」を動き（モーション）について具体化する。
モーションは紙芝居感を消し、情報理解を助け、量産に耐え、コードで直せるものでなければ通さない。
平均点では通さない。重大欠陥は1つでもblocker（12のSeverityに従う）。

評価対象:

- MotionDemo（量産前に1モーション設計を検証する単体デモ）
- 本番ショット（Scene Planに組み込まれた実ショット）

---

## 2. 7軸 評価モデル

各軸は 1〜5 のルーブリックを持つ。1=不可、5=理想。
**合格閾値は原則 各軸 ≥3、かつ Readability・Consistency・Code Operability は ≥4。**
いずれかの軸が 1 の場合は自動的に blocker（S4以上）。

| 軸 | 何を見るか | 合格閾値 | blocker条件 |
|---|---|---|---|
| Beauty（美しさ） | 構図・余白・配色・質感が破綻なく美しいか | ≥3 | 破綻=1 |
| Dynamism（動きの意味） | 動きに物語・情報的な意味があるか、無駄な動きでないか | ≥3 | 意味のない常時アニメ=1 |
| Readability（可読性） | 文字・図・ナレ理解を動きが邪魔しないか | ≥4 | 読めない=1 |
| Consistency（統一性） | 素材の質感・速度・イージング・トーンが他ショットと揃うか | ≥4 | 浮く/別作品に見える=1 |
| Production Efficiency（量産効率） | 量産に耐えるか、1ショットあたり手間が許容内か | ≥3 | 1ショットに人手過大=1 |
| Code Operability（コード操作性） | Claude Code/Codexが props・preset で直せるか | ≥4 | ハードコードで直せない=1 |
| Asset Reusability（素材再利用性） | Asset Factoryから選べる/Scene Planで再利用できるか | ≥3 | 使い捨て専用=1 |

### 2.1 Beauty（美しさ）

観点: 構図、余白、配色、コントラスト、質感、ブラー/ノイズの自然さ、AIっぽさの少なさ（自然・実写志向）。

- 1: 構図破綻、色濁り、明らかなAIアーティファクト。
- 2: 見られるが安っぽい。素材の解像感不足。
- 3: 標準的に美しい。違和感なし。
- 4: 意図した質感が一貫し、画面として品がある。
- 5: そのまま見せ場に使える完成度。実写的説得力。

重点チェック: 美しいか / AIっぽさが抑えられているか。

### 2.2 Dynamism（動きの意味）

観点: その動きが「なぜ動くのか」を説明できるか。注意誘導、因果、時間経過、強調のいずれかに紐づくか。意味のない常時揺れ・常時回転は減点。

- 1: 全要素が意味なく常時動く（モーション酔い）。
- 2: 動きはあるが装飾的で情報に寄与しない。
- 3: 主要な動きに意味がある。
- 4: 動きが視線と理解を正しく導く。
- 5: 動き自体が物語を語る（30 motion grammar を的確に適用）。

重点チェック: 動きに意味があるか。

### 2.3 Readability（可読性）

観点: オンスクリーンテキスト/図表の可読性、動きが読解を阻害しないか、保持時間が十分か、セーフエリア内か。

- 1: 文字が読めない/動きで読解不能。
- 2: 読めるが速すぎる・かぶる。
- 3: 読める。保持時間十分。
- 4: 動きが理解を助ける（出現順が論理に一致）。
- 5: 情報設計とモーションが一体で、初見で理解できる。

重点チェック: 情報理解を邪魔しないか。

### 2.4 Consistency（統一性）

観点: 質感・速度・イージング・トーン・パレットが他ショット/エピソード基準と揃うか。素材出自（Midjourney/Lottie/Runway）が混在しても統一感があるか。

- 1: 別作品に見える。素材がバラバラ。
- 2: トーンが部分的にずれる。
- 3: 概ね統一。
- 4: 速度・イージング・パレットがプリセットで統一。
- 5: シリーズ全体で同一の「PDらしさ」。

重点チェック: 素材の質感が統一されているか。

### 2.5 Production Efficiency（量産効率）

観点: 1ショットあたりの生成回数・人手・レンダー時間が許容内か。Runwayは見せ場のみに限定できているか。紙芝居（静止画切替だけ）からの脱却が低コストで実現できているか。

- 1: 1ショットに過大な人手/高額生成。
- 2: 量産すると破綻する手間。
- 3: 量産可能だが最適化余地あり。
- 4: presetで横展開でき、手間が小さい。
- 5: ほぼ自動で量産でき、Runway消費も最小。

重点チェック: 紙芝居感が減るか / 量産に耐えるか。

### 2.6 Code Operability（コード操作性）

観点: Claude Code/Codexが props・preset を変えるだけで修正できるか。マジックナンバーのハードコードがないか。差分が小さく安全か（既存を壊さない）。

- 1: 値がハードコードでコードから直せない。
- 2: 一部propsだが多くが固定値。
- 3: 主要パラメータがprops化。
- 4: preset化され、名前付きで選択・上書き可能。
- 5: props/presetだけで全調整可能、テスト容易、再現性あり。

重点チェック: Claude Code・Codexが props・preset で修正できるか。

### 2.7 Asset Reusability（素材再利用性）

観点: Asset Factory（棚）から選択できるか、manifestに登録され usedIn が記録されるか、Scene Planの複数箇所で再利用できるか、使い捨てでないか。

- 1: この1ショット専用、再利用不可。
- 2: 理論上再利用可だが未登録。
- 3: manifest登録済みで再利用可能。
- 4: 複数シーンで実際に再利用、usedIn記録あり。
- 5: 汎用プリセット/部品として棚に常備、生成回数削減に寄与。

重点チェック: Asset Factoryから選択できるか / Scene Planで再利用できるか。

## 3. SFX同期チェック（横断・必須）

モーションは音と合っていなければ「動いているだけ」になる。以下は全ショット共通の追加判定（Readability/Dynamismに影響、不合格はS3以上）。

- [ ] キーモーション（出現・転換・強調）がElevenLabs SFXまたはBGMアクセントと**フレーム単位で同期**している。
- [ ] SFXが過剰でナレを邪魔していない（12 Audio Gate「no distracting SFX」と整合）。
- [ ] 同期に使ったSFXは**再利用素材**であり、使い捨て新規生成を増やしていない。

重点チェック: SFXと同期しているか。

## 4. MotionDemoゲート と 本番ゲート

### 4.1 MotionDemoゲート（量産前の単体検証）

目的: 1つのモーション設計を「棚に入れて良いか」判定する。ここを通ったものだけ量産に使う。

合格条件:

- 7軸すべて閾値以上（特に Code Operability ≥4、Consistency ≥4）。
- props/preset の一覧が文書化され、Claude Code/Codexが変更点を再現できる。
- 少なくとも2つの異なる入力（テキスト長・画像）で破綻しないことを確認。
- SFX同期チェックを満たす。
- Asset Factoryへの登録方法（ID, usedIn初期値）が定義されている。

不合格時: 棚に入れない。preset修正→再デモ。本番には絶対に使わない。

### 4.2 本番ショットゲート（Scene Plan組み込み後）

目的: 実エピソードのショットとして公開可否を判定する。12のVisual/Edit Gateに先行する細分ゲート。

合格条件:

- 使用したモーションは**MotionDemoゲートを通過済み**の棚素材であること（未検証モーションの本番直行は不可）。
- 7軸すべて閾値以上。Readability ≥4 を厳守（情報を邪魔しない）。
- 当該ショットの素材がmanifestに記録され、usedInに当該シーンが追記される。
- SFX/BGM同期チェック合格。
- Runway使用は「見せ場」と明記されたショットのみ（CANON: Runwayは見せ場動画のみ）。
- 既存ショットと統一感がある（Consistency ≥4）。

不合格時: 12のSeverityで差し戻し（S3=公開前修正必須、S4=工程差戻し）。

### 4.3 12との関係まとめ

- 12 = 全工程ゲートの上位（Topic〜Package）。
- 本書 = 12の Visual Gate / Edit Gate のうち「モーション」を深掘りする下位ゲート。
- 本書で blocker（軸=1, またはSFX同期S3+）が出たら、12のVisual/Edit Gateも自動的に不合格とする。
- 本書はSeverityを新設せず、12のS0〜S5を流用する。

## 7. 運用フロー（要約）

1. モーション設計 → **MotionDemoゲート**で7軸+SFX採点 → 合格のみ棚（Asset Factory）へ登録（ID/usedIn/preset一覧）。
2. Scene Planは棚のモーションを選択して組み込む（新規設計は原則しない＝Asset First）。
3. 本番ショット完成 → **本番ショットゲート**で再採点 → usedIn追記。
4. blocker発生 → 修正テンプレ → props/preset修正を最優先（新規生成は最後）。
5. 12のVisual/Edit Gateへ引き継ぎ（本書の合否を含めて最終判定）。
