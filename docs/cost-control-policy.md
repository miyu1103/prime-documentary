# Cost Control Policy（コスト統制ポリシー）

ステータス: 設計（draft）

前提・仮定:

- 本書は CANON に従う。前提ツールは登録済みのみ: Claude(Pro/Max), ChatGPT(Plus/Pro), GitHub, Runway, ElevenLabs, Midjourney, SUNO, LottieFiles(無料登録)。
- **新規有料ツール・新規素材サブスクは前提にしない。** クレジットカードが必要な「無料トライアル」も回避する。
- 秘密情報（APIキー・トークン・Cookie等）はコードやdocsに書かない。
- ツール役割（固定）: Midjourney=静止画/パーツ, Runway=見せ場動画のみ, ElevenLabs=ナレ+SFX, SUNO=BGM, Lottie=手動DLしたローカルJSONのみ。
- 中核方針は **Asset First**＝先に大量素材を貯め、再利用で生成回数を減らす。
- 本書はコスト面の運用規約であり、品質判定は `docs/motion-quality-gate.md` と `docs/12_QUALITY_GATES_AND_ACCEPTANCE.md` に従う。

---

## 1. 基本原則（明記事項）

1. **新規有料ツールを前提にしない。** 設計・実装・提案は、上記の登録済みツールだけで成立させる。
2. **無料・ローカル・OSSを優先する。** 同じ結果が無料/ローカル/OSSで得られるなら、有料生成を選ばない。
3. **Asset First。** 生成より先に「棚（Asset Factory）」を探す。棚にあれば生成しない。
4. **Runwayは見せ場だけ。** 通常ショットはMidjourney静止画＋コード（Remotion）モーション＋Lottieで作る。Runwayはエピソードの見せ場に限定。
5. **再利用を最大化する。** すべての素材はmanifestに登録し、`usedIn` で使用箇所を記録する。
6. **重複生成を検知する。** 生成前にプロンプト/用途の近い既存素材がないか必ず確認する。
7. **追加課金は例外。** 追加有料ツール・大量生成は、後述のテンプレで「なぜ既存で代替不可か」を説明してからのみ検討する。
8. クレカ必須の無料トライアルは使わない（解約忘れ・意図せぬ課金・BANリスク回避）。

---

## 2. ツール別「再利用を最大化する」運用

共通ルール: 生成の前に「棚から探す → 無ければ再利用可能な形で生成 → manifest登録（ID/provenance/hash）→ usedIn記録」。

### 2.1 Midjourney（静止画 / パーツ）

- 用途: 背景、人物以外のオブジェクト、UIパーツ、テクスチャ、図版素材。
- 再利用ルール:
  - 1枚を1ショット専用にしない。背景・パーツは複数シーンで使い回せる構図・余白で作る。
  - バリエーションは「必要数だけ」。同義の再生成を避ける（重複検知）。
  - manifestに `prompt`, `seed/style`, `usedIn[]` を記録。同一プロンプトの再生成前にusedInを確認。
  - 切り抜き/タイル化したパーツは部品として棚に常備し、Remotionで合成する。
- 避ける無駄: 「なんとなく良い別案」を量産する、同じ被写体を毎回作り直す。

### 2.2 Runway（見せ場動画のみ）

- 用途: エピソードの山場・印象づけショットのみ。本数を事前に上限化する。
- 再利用ルール:
  - 生成前に「本当に見せ場か」を判定（見せ場でなければMidjourney静止画＋コードモーションで代替）。
  - 生成したクリップは尺・カット違いで再利用できるよう長めに確保し、manifest登録＋usedIn。
  - 同一見せ場の微差し替えはコード側（トリミング/速度/イージング）で対応し、再生成しない。
- 避ける無駄: 通常ショットをRunwayで作る、見せ場の細部調整のたびに再生成する。

### 2.3 ElevenLabs（ナレーション + SFX）

- 用途: マスターナレーション、効果音。
- 再利用ルール:
  - SFXはライブラリ化する。汎用SFX（出現音、転換音、ホイッシュ、UIクリック等）は1度作って棚へ。
  - manifestに `type=sfx`, `usedIn[]` を記録。新規SFXを作る前に棚の汎用SFXで足りないか確認。
  - ナレーションは台本確定後にのみマスター生成（ドラフトは安価運用、確定前の作り直しを避ける）。
- 避ける無駄: 同種SFXを毎エピソード作り直す、台本未確定でマスターナレを生成して作り直す。

### 2.4 SUNO（BGM）

- 用途: BGM。
- 再利用ルール:
  - 生成したBGMは音楽ライブラリ（rights-tracked）に登録し、ムード/テンポでタグ付けして再利用。
  - 1エピソード専用にしない。ムード別の定番トラックを棚に育てる。
  - manifestに `mood`, `bpm`, `usedIn[]` を記録。新規生成前にライブラリを検索。
  - 整理: 重複ムードのトラックは代表を決め、残りはアーカイブ（生成し続けない）。
- 避ける無駄: 似たムードのBGMを毎回生成、ライブラリ未整理で探せず再生成。

### 2.5 LottieFiles（手動DLしたローカルJSONのみ）

- 用途: 軽量UIアニメ、アイコン、図解モーション。
- 再利用ルール:
  - **無料素材のみ**を手動DLし、ローカルJSONとしてリポジトリ/ローカル棚に保存して使う（APIや有料プランは前提にしない）。
  - 色・速度はコード側（props）で調整して再利用。1つのJSONを複数用途に展開。
  - 権利メモ（ライセンス種別）をmanifestに記録（商用可否を必ず確認）。
- 避ける無駄: 同等の無料JSONがあるのに有料素材を検討する、ライセンス未確認で使う。

## 3. 重複生成の検知（棚から探してから生成）

生成リクエスト前に必ず実施する「探索→判断」手順:

1. **棚検索**: manifestを `用途/プロンプト類似/mood/type` で検索し、再利用可能な既存素材を列挙。
2. **再利用判定**: 既存素材＋コード調整（props/preset、トリミング、色替え、速度）で要件を満たせるか判定。
3. **満たせる**→ 生成しない。既存素材の `usedIn[]` に新しい使用箇所を追記。
4. **満たせない**→ 「なぜ既存で不可か」を1行記録してから最小限だけ生成。
5. **生成後**→ manifestへ `id / provider / prompt or params / hash / provenance / usedIn[]` を登録。

重複検知の指標（運用で監視）:

- 同一/類似プロンプトの生成回数。
- `usedIn` が空のまま放置されている素材（=使い捨て予備軍）。
- 同ムード/同type素材の本数（過剰なら整理）。

## 4. 追加課金を検討する条件（例外プロセス）

以下を**すべて**満たす場合のみ、追加有料ツール/大量生成を「検討」できる（=即採用ではない）。

- 既存の登録済みツール＋OSS/ローカルでは**品質ゲートを通せない**ことが具体例で示せる。
- 効果（品質/時間/再利用）が定量・定性で説明できる。
- コスト上限・idempotency・課金チェックの方針がある（既存の課金安全ルールに整合）。
- クレカ必須の無料トライアルではない、またはBAN/解約リスクが管理できる。
- メモリのコスト方針に従い、超過課金になる場合は日本語で停止・確認する。

検討しても、原則は「前提にしない・候補メモのみ」。

### 4.1 追加有料ツール提案テンプレ（必須）

追加有料ツールを提案する場合は、必ず「なぜ既存で代替不可か」を説明する。下記テンプレを埋めない提案は却下。

```
# 追加有料ツール提案
提案ツール: __________（用途: __________）

## なぜ既存で代替不可か（必須）
- 試した既存手段: Midjourney / Runway / ElevenLabs / SUNO / Lottie / OSS / Remotionコード
- 具体的に何が不足したか（品質ゲートのどの軸で落ちたか）: __________
- コード/props/プリセットでも解決できない理由: __________

## 効果と代替案
- 期待効果（品質/時間/再利用、定量があれば数値）: __________
- 代替案（やらない場合の回避策）: __________

## コスト安全
- 想定コスト/上限: __________
- idempotency/課金チェック: __________
- クレカ必須トライアルか: yes/no（yesなら原則却下）

## 結論
[ ] 候補メモのみ（前提にしない）  [ ] 例外採用を人間承認に上げる
```

### 4.2 将来候補（前提にしない・候補メモのみ）

- Luma / Kling / Pika（動画生成）、Rive（インタラクティブモーション）等。
- これらは**前提にしない**。役割が現状ツールで埋まらないと §4 で実証された場合に限り、候補メモとして検討対象に挙げるのみ。現時点で設計・実装の前提に組み込まない。

## 5. Asset First による生成回数削減効果（定性）

数値は環境依存のため固定値は約束しないが、Asset Firstの定性効果は以下。

- **背景/パーツ（Midjourney）**: 1枚を複数シーンで再利用するほど、エピソードあたりの新規静止画生成は逓減する。シリーズが進むほど棚が育ち、新規生成比率が下がる。
- **見せ場（Runway）**: 「見せ場だけ」に限定し通常ショットをコードモーションで賄うことで、高コスト動画生成の本数を構造的に抑制。
- **SFX（ElevenLabs）**: 汎用SFXライブラリ化により、同種効果音の再生成をほぼゼロに近づける。
- **BGM（SUNO）**: ムード別定番トラックの再利用で、エピソードごとの新規BGM生成を削減。ライブラリ整理で「探せず再生成」を防止。
- **モーション（Remotion preset + Lottie）**: MotionDemoで検証済みのpreset/JSONを使い回し、ショットごとの新規モーション設計をなくす。
- **総合**: 棚の成熟に比例して「新規生成/エピソード」が下がり、人手レビュー（motion-quality-gateの再採点）も再利用素材分は省略できる。最大の削減源は「生成の前に棚を探す」運用の徹底。
