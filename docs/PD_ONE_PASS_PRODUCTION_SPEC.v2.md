**Status:** BINDING for every episode from **EP19 onward**. Supersedes v1 (v1 remains the record EP17/EP18 were built to). Consolidates decisions/0002, 0004–0007 + `scripts/check_final_acceptance.py`.

## THE ONE RULE
> **"Done" ≠ validator PASS. "Done" = the independent acceptance script measures the real `final.mp4` (+ provenance) and ALL hard gates pass.** The producing agent may NOT hand-write its own quality gate. Self-reported QC is rejected (CLAUDE.md invariant 13, rule 17). A new revision re-runs the full gate.

Final gate command (must exit 0 before "package_ready"):
```
./.venv/Scripts/python.exe scripts/check_final_acceptance.py <NN> --json
```
The runtime band is read from the episode's duration profile (standard 11.5–12.5 / **mid 27–33** / feature 55–65), NOT hardcoded. **EP19/EP20 = mid (target 30:00).**

## E. CODEX DETERMINISM (why the handoff keeps drifting, and the fix)
Codex builds from the design doc **standalone** (it does not read CLAUDE.md or memory). When the result "isn't what I wanted," the cause is an under-specified doc, not Codex. So the per-episode handoff MUST be written to leave **zero interpretation room**:
- **Numbers, not adjectives.** Never "fast / dramatic / subtle." Always the exact value: start/end frame, easing type (`spring{damping,stiffness}` or `Easing.out(Easing.cubic)`), zoom % per shot, transition length in seconds, BGM duck level in LUFS, caption cue boundaries, shot order with per-shot seconds.
- **Per-asset, per-shot tables.** The `shotlist.<v>` lists every shot: id, source file/prompt id, seconds, motion (Ken Burns vector or parallax), transition in/out, factory layer, caption span. No "etc." / no "and so on."
- **One image = one prompt.** `ai_prompts.<v>` has a numbered, self-contained prompt per hero still (subject, composition, lens, light, palette, mood, negative list, upscale target ≥3840). Same for each thumbnail variant.
- **State the value even when obvious** (resolution, fps, voice id, crf, color space, audio bitrate) so Codex never guesses a default.
- **The gate is the contract, not the chat.** Codex must run `check_final_acceptance.py <NN>` itself and not declare done until exit 0. The acceptance script is what makes "thought-through" survive contact with Codex — if a behavior matters, it must be BOTH written as an exact spec here AND enforced by a measurable gate (rows 1–16). A requirement with no gate WILL drift; add the gate.
- **Acceptance restated at the top of every Codex prompt**: the exact rows the render must satisfy + the verify command, so Codex self-checks before returning.
